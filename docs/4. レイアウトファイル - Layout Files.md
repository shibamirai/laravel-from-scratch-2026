# レイアウトファイル

Blade コンポーネントとレイアウトについて話しましょう — 繰り返しマークアップを再利用可能なパーツに抽出し、データを渡して属性をマージする方法についてです。
これにより、利用者がスタイルをクリーンに上書きできるようになります。
これがビューの保守性を高め、作業を楽しくする秘訣です。

## コンポーネントとレイアウトの基本

> Bladeコンポーネントは、ビュー間で再利用可能な HTML フラグメントと考えてください。

レイアウトとは、ページ（ナビゲーション、ヘッダー、スクリプトなど）のラッパーとして使用するコンポーネントに過ぎません。
ページ固有のコンテンツは、そのコンポーネントのスロットに配置されます。

これが私が愛用しているシンプルな静的ページのルートヘルパーです（連絡先や会社概要ページに最適）：

```php
// routes/web.php
Route::view('/about', 'about');
Route::view('/contact', 'contact');
```

これにより、ほとんど静的なページ用に小さなコントローラーを書く手間が省けます。

## レイアウトコンポーネントの作成

レイアウトファイルを作成し、共有部分（ヘッダー、ナビゲーション、フッター）をそこに移動します。
その後、カスタムタグを使用してビューでそれを利用します。

レイアウトコンポーネントファイルの例：

```html
<!-- resources/views/components/layout.blade.php -->
<!doctype html>
<html data-theme="dark">
<head>
  <meta charset="utf-8">
  <title>{{ $title ?? 'My App' }}</title>
</head>
<body>
  <x-nav /> <!-- your nav component -->
  <main class="max-w-3xl mx-auto px-6 mt-8">
    {{ $slot }}
  </main>
</body>
</html>
```

ページ内で使用するには、ページコンテンツをコンポーネントタグで囲みます：

```html
<!-- resources/views/about.blade.php -->
<x-layout title="About Us">
  <h1>About</h1>
  <p>...content...</p>
</x-layout>
```

その `{{ $slot }}` が、ページの固有マークアップが挿入される箇所です。

## Props: コンポーネントにデータを渡す

コンポーネントが動的データ（ページタイトルなど）を必要とする場合、コンポーネントファイルの先頭でプロップを宣言します。
最も簡単な方法は `@props` です：

```html
@props(['title' => 'Laracasts'])

<!doctype html>
<html>
<head>
  <title>{{ $title }}</title>
</head>
<body>
  {{ $slot }}
</body>
</html>
```

ビューが `title` を渡さない場合、デフォルトの `Laracasts` が使用されます。

## Slots: ページコンテンツの配置場所

`<x-layout> ... </x-layout>` を使用すると、タグ間のすべてがコンポーネントのスロットとなり、`{{ $slot }}` が記述された箇所に挿入されます。
これによりレイアウトは汎用性を保ちつつ、個別のページコンテンツを受け入れることが可能になります。

## Attributes Vs Props そしてクラスのマージ

コンポーネントに渡すもので、プロップとして宣言されていないものはすべて、そのコンポーネントの属性バッグの一部となります。
`$attributes`（およびそのヘルパー）を使用して、それらを公開し、ルート要素にマージしてください。

例：カードコンポーネント

```html
<!-- resources/views/components/card.blade.php -->
@props([])

<div {{ $attributes->merge(['class' => 'bg-gray-100 p-6 text-center']) }}>
  {{ $slot }}
</div>
```

これを使って、外部からクラスを上書き／拡張する：

```html
<x-card class="max-w-400 mx-auto">Contact form placeholder</x-card>
```

最終的にレンダリングされる要素には、カードのデフォルトクラスとユーザーが指定した `max-w-400 mx-auto` の両方が含まれます。

## 再利用可能な UI の作成（アイデアカードの例）

マークアップを複製する代わりに、繰り返し使用されるUIブロックをコンポーネントとして抽出します。

コンポーネント：

```html
<!-- resources/views/components/idea-card.blade.php -->
@props(['href' => '#'])

<a {{ $attributes->merge(['class' => 'card p-4 bg-base-100 hover:shadow-md block']) }} href="{{ $href }}">
  {{ $slot }}
</a>
```

使い方：

```html
<x-idea-card href="{{ route('ideas.show', $idea) }}">
  <h3 class="font-semibold">{{ $idea->title }}</h3>
  <p class="text-sm text-muted">{{ $idea->description }}</p>
</x-idea-card>
```

より小さなパーツ（ナビゲーション、フッター、フォームコントロール）が必要な場合は、それらもコンポーネント化してください。
構成が大事です。

## 小さなワークフローのコツ

- 可能な限りコンポーネント/ビューで名前付きルートを使用してください（例: `route('profile.edit')`）。これによりリファクタリングでリンクが壊れるのを防げます。
- 共有スタイルはスタイルシートに記述し、インラインスタイルは簡易な実験用のみに留めます。
- エディターが動的プロパティ（リレーションシップコレクションなど）について警告する場合、開発体験を円滑にするため、ヘルパースタブ（Laravel IDE Helper / Laravel Idea）を生成するか、モデルに PHPDoc の `@property` アノテーションを追加してください。

## 簡単なまとめ

- 単純な静的ページには `Route::view()` を使用します。
- レイアウトコンポーネントを作成し、ページコンテンツをスロットに配置します。
- 動的データには `@props([...])` でプロパティを宣言します。
- 未登録の属性は `$attributes` 経由で利用可能 — デフォルト値とコンシューマークラスを結合するには `->merge()` を使用。
- 繰り返しマークアップ（カード、ナビゲーション、フォーム）は再利用性と明瞭化のためコンポーネントに抽出。

これでコンポーネントとレイアウトの基礎は十分です。
これらのパターンに慣れれば、ビューのメンテナンスやスタイリングが格段に楽になります。
