# Vite によるフロントエンドアセットバンドリング

フロントエンドアセットバンドリングについて話しましょう ― なぜいつまでも CDN に頼ってはいけないのか、Laravel アプリで Vite と Tailwind(と DaisyUI) を束ねる方法、そしてホットリロードと運用ビルドを行うのに使用するコマンドについてです。
実践的な内容になるよう、私が実際にやっているコツをお見せします。

## バンドリングとは何か、なぜ必要か

> バンドリングとは、ローカル環境にある CSS や JavaScript（と依存関係）をコンパイルして最適化し、運用環境で実際にブラウザがロードするファイルにまとめることです。

学習中であれば必要なものをすべて CDN から参照しても構いません。
ですが実際のアプリではローカルでビルドしたものによる速度、制御、信頼性が必要になります：必要なスタイルだけを取り出した CSS、サイズを減らした JS、開発時のソースマップ、修正中のホットリロード、そしてデプロイ用のファイルの生成です。

## Vite と Laravel

Laravel では最初から Vite が使える状態になっています。
Vite はソースコードを監視し、束ね、開発時のホットリロードを提供します。
運用では、アセットを最適化しサイズを減らして出力します。

こちらが新規 Laravel プロジェクトに入っている最小限の Vite セットアップ (vite.config.js) です：

```javascript
import { defineConfig } from 'vite';
import laravel from 'laravel-vite-plugin';

export default defineConfig({
  plugins: [
    laravel({
      input: ['resources/css/app.css', 'resources/js/app.js'],
      refresh: true, // バックエンド／テンプレートが変更されたときにブラウザを自動でリフレッシュする
    }),
  ],
});
```

この `refresh: true` は便利です ― Blade テンプレートやその他サーバー側でのアセットの変更時に再読み込みをするよう Vite に伝えます。

## Tailwind CSS と DaisyUI (ローカル版)

npm で Tailwind（必要であれば DaisyUI も）をローカルにインストールすると、コンパイルして不要なクラスを減らすことができます。

```bash
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p

# オプション: UI コンポーネントライブラリーのインストール
npm install daisyui
```

tailwind.config.cjs (または tailwind.config.js) で DaisyUI を有効化し、blade/js ファイルの位置を示して Tailwind が対象とするファイルを伝えます。

```javascript
module.exports = {
  content: [
    './resources/**/*.blade.php',
    './resources/**/*.js',
    './resources/**/*.vue',
  ],
  theme: {
    extend: {},
  },
  plugins: [
    require('daisyui'),
  ],
};
```

resources/css/app.css で Tailwind の各レイヤー（と DaisyUI）をインポートします。

```css
@tailwind base;
@tailwind components;
@tailwind utilities;

/* カスタム変数やユーティリティはここに */
:root {
  --color-primary: 34 197 94; /* Tailwind v4 スタイルの RGB 変数の例 */
}
```

## Blade でビルド済みファイル読み込む

コンパイルされたアセットをインクルードするには Vite ディレクティブを使用します。
Vite は開発中には HMR アセットを、運用ではビルド済みファイルを提供します。

```html
<!-- resources/views/layouts/app.blade.php -->
<!doctype html>
<html>
  <head>
    @vite(['resources/css/app.css', 'resources/js/app.js'])
  </head>
  <body>
    {{ $slot }}
  </body>
</html>
```

このディレクトリ１つで開発環境と運用環境の両方を扱えます。

## 開発時のワークフロー: ホットリロードとビルド

ローカルの開発環境では dev/watch モードで Vite を実行します。
これはメモリ上でアセットをコンパイルし、ホットリロードを提供します。

```bash
npm install   # まず依存関係をインストール（１度だけ）
npm run dev   # dev モードで vite を実行（ホットリロード）
```

このプロセスを停止すると Vite は監視をしなくなり、CSS/JS の変更はブラウザに反映されません。
もし app.css や resources/js/app.js を修正したのに更新されない場合は、`npm run dev`（または vite）が実行されているか確認してください。

本番用にビルドする場合は：

```bash
npm run build
```

Vite は最適化されたファイルを `public/build` フォルダ（またはセットアップで指定した場所）に出力します。
これらを本番環境でデプロイします。

## 例：簡単な JS の確認

JS がちゃんとバンドルされて実行されるか確認したければ、resources/js/app.js に何か処理を追加してみましょう。

```javascript
console.log('app.js loaded');
```

`npm run dev` を実行してからブラウザのコンソールを開くとこのログが出力されているはずです。
`npm run build` の実行後でも、コンパイルされたファイルの中でこのメッセージは残っているでしょう。

## Tailwind の JIT / Purge について

Tailwind は Blade と JS をスキャンし、使用されていないものを本番用ビルドから取り除きます。
クラスを使用している場所すべて（ビュー、コンポーネント、Vue/React ファイル）が tailwind.config で指定されているか確認し、最終的なファイルが小さいままとなるようにしましょう。

## アセットのデプロイ

本番環境には２つのアプローチ方法があります。

- ローカルでビルド (`npm run build`) し、`public/build` にコミット/配置します（またはサーバと同期させます）。作成するファイルを完全に制御する方法です。
- CI/CD パイプラインの中でビルドします（チーム開発で推奨されます）。デプロイするとサーバまたは CI 環境でリリース前に `npm ci && npm run build` が実行されます。

どちらも正しい方法ですが、チーム開発では CI でのビルドの方が一般的です。

## 素早い問題解決のための効果的な手法

もしスタイルが突然消えたり変更が反映されなくなったら、このような原因であることが多いです：`npm run dev` を実行し忘れているか、古いキャッシュが残っているか、`vite` ディレクティブがレイアウトにないかです。
Vite を再実行し、キャッシュをクリアし、コンソールやネットワークを確認しましょう。

本番環境で Tailwind が効いていないところがあった場合は、そのクラスを使っているところが `content` の設定に含まれているか確認します。

## TL;DR（私が実際にやっていること）

1. ローカルで Tailwind/DaisyUI をインストール(npm)
1. Blade で `@vite([...])` を使用し、ローカルの CSS/JS をインクルード
1. 開発環境では `npm run dev` を実行し、ホットリロードできるようにする
1. デプロイ時は `npm run build` を実行し、アセットを最適化
1. 不要な設定が除かれるように、Tailwind の `content` にすべての Blade/JS/Vue ファイルが含まれているか確認

必要であれば、そのままプロジェクトに使用できる最低限のファイルのセットをお渡しできます：vite.config.js、tailwind.config.cjs、resources/css/app.css、resources/js/app.js、それからアプリにそのままコピーペーストできる Blade レイアウトです。
