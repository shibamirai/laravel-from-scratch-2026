# [DaisyUI についてちょっと寄り道](https://laracasts.com/series/laravel-from-scratch-2026/episodes/13)

見た目の改善について数分間話しましょう — アプリを現実世界に溶け込ませるための、手早く実用的な調整です。
Tailwind を導入し（おそらく既に導入済みでしょう）、その上に DaisyUI を載せてナビゲーション、カード、ボタン、フォームを素早く組み立て、さらに Blade コンポーネントをいくつか抽出してマークアップを整理する方法を示します。

## DaisyUI と Tailwind

> DaisyUI は Tailwind CSS を基盤として構築されたコンポーネントライブラリです。

DaisyUI は Tailwind スタイルのコンポーネント（ボタン、カード、ナビゲーションバー、モーダルなど）をすぐに使える状態で提供するため、数分で UI のプロトタイプを作成できます。
通常の Tailwind ビルドプロセスを通じて使用できるほか、迅速なプロトタイプ作成にはドキュメントで提供される CDN/スニペットを利用できます。

基本的な考え方は以下の通りです：Tailwind（または Vite でビルドした CSS）と DaisyUI を組み込み、html タグに data-theme 属性を設定してビジュアルテーマを切り替えます。

```html
<!-- 例: ルート HTML 要素にテーマを設定する -->
<html data-theme="dark">
  ...
</html>
```

DaisyUI には多数のテーマ名（dracula、coffee、business、dark、lightなど）が同梱されています。
その値を切り替えるだけで、即座に異なる見た目に変更できます。

## ナビゲーションバーをすばやく追加する

素早く試したい場合は、DaisyUI のサンプルからナビゲーションバーのスニペットを取得し、レイアウトに貼り付けてください。
貼り付けると、即座にスタイルが適用されたナビゲーションバーが完成します。
私はそのマークアップをコンポーネントに移動し、レイアウトをクリーンに保つようにしています。

以下に、Blade コンポーネントに抽出してレイアウト内でレンダリングする大まかな方法を示します。

resources/views/components/nav.blade.php に Blade コンポーネントビューを作成します：

```html
<!-- resources/views/components/nav.blade.php -->
<nav class="bg-base-200">
  <div class="container mx-auto px-6">
    <div class="flex justify-between items-center py-3">
      <a href="{{ route('ideas.index') }}" class="text-lg font-bold">Idea</a>

      <div class="flex gap-4 items-center">
        <a href="{{ route('ideas.index') }}" class="link">All Ideas</a>
        <a href="{{ route('ideas.create') }}" class="btn btn-sm">New Idea</a>
      </div>
    </div>
  </div>
</nav>
```

これをレイアウトからレンダリングしてください：

```html
<!-- in resources/views/layouts/app.blade.php -->
<x-nav />
<main class="max-w-3xl mx-auto px-6 mt-8">
  @yield('content')
</main>
```

ナビゲーションをコンポーネントに分割することで、レイアウトファイルを短く保ち、アプリ全体のナビゲーションを調整する場所を1か所にまとめられます。

## カード、グリッド、再利用可能なコンポーネント

再利用可能な「アイデアカード」コンポーネントを作成するのが好きです。
そうすれば、インデックスや関連リスト、その他の場所でも同じ UI をレンダリングできます。
コンポーネントはマークアップを DRY (Don't Repeat Yourself - 繰り返しを避ける) に保ちます。

属性を受け取り、柔軟なコンテンツのためにスロットを使用するシンプルなアイデアカードコンポーネントです：

```html
<!-- resources/views/components/idea-card.blade.php -->
<a {{ $attributes->merge(['class' => 'card p-4 bg-base-100 hover:shadow-md block']) }} >
  {{ $slot }}
</a>
```

インデックスビューで次のように使用してください：

```html
<div class="grid md:grid-cols-2 gap-6">
  @foreach($ideas as $idea)
    <x-idea-card href="{{ route('ideas.show', $idea) }}">
      <h3 class="font-semibold">{{ $idea->title }}</h3>
      <p class="text-sm text-muted">{{ $idea->description }}</p>
    </x-idea-card>
  @endforeach
</div>
```

タイトルや説明をプロパティとして渡すことを好む場合は、コンポーネントのプロパティを作成できます。
ただし、内部のマークアップが使用ごとにわずかに異なる場合には、スロットが便利です。

## フォームコンポーネントとバリデーションのスタイリング

DaisyUI は洗練されたフォーム制御クラスを提供します。
バリデーションエラーを強調表示したい場合は、Blade の @error ディレクティブと DaisyUI のクラスを組み合わせて、必要に応じて「error」スタイルを追加してください。

```html
<textarea name="description"
          class="textarea w-full @error('description') textarea-error @enderror"></textarea>

@error('description')
  <p class="text-sm text-error mt-1">{{ $message }}</p>
@enderror
```

DaisyUI には、典型的なバリデーション状態にうまく対応する意味の分かりやすいユーティリティクラス（text-error、textarea-error、btn-primary、btn-outlineなど）が用意されています。

## テーマで遊ぶ

DaisyUI は標準でテーマをサポートしているため、デザイン中にテーマを切り替えてみてください。
html タグの data-theme を変更し、ページを更新すると、すべてのコンポーネントでカラーパレットが変化します。
これは構築中に視覚的な選択肢を素早く探るのに最適な方法です。

## 小さなワークフローのコツ

実験段階であれば、CDN または Tailwind Play スタイルの設定が最速です。
本番環境では、最適化された CSS とテーマ制御を得るために、Tailwind ビルド（Vite）と DaisyUI をプラグインまたはローカルパッケージとして使用することをお勧めします。

UI のパーツをコンポーネント化する際は、小さく焦点を絞ったものにしましょう：ナビゲーションコンポーネント、再利用可能なカードコンポーネント、フォームフィールドコンポーネントなどです。
これにより再利用が容易になり、スタイルの一貫性を保てます。
