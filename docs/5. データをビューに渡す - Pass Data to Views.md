# [データをビューに渡す](https://laracasts.com/series/laravel-from-scratch-2026/episodes/5)

ビューへのデータ渡しについて話しましょう — 私が日常的に使う、小さくて実用的なパターン：Route::view ショートカット、ルートクロージャーからのビュー返却、クエリ文字列の読み取り、そして Blade の自動エスケープがなぜあなたの最高の味方なのかについてです。

## データをビューに渡す

> ビューを返す際に連想配列を渡すと、その配列の各キーがビュー内で利用可能な変数として抽出されます。

これが基本ルールで、非常に便利です。
ホームページに挨拶文を表示したい場合、Route::view ショートカットと第三引数を使って実現できます：

```php
// routes/web.php
Route::view('/', 'welcome', [
    'greeting' => 'Hello',
    'person' => 'John',
]);
```

次に、resources/views/welcome.blade.php でそれらの変数を出力できます:

```html
<p>{{ $greeting }}, {{ $person }}.</p>
```

ホームページにアクセスすると、“Hello, John.” と表示されます。

## ルートクロージャーからビューを返す

リクエストデータを最初に計算または読み取る必要がある場合、ルートクロージャからビューを返し、第二引数をデータ配列として渡します：

```php
// routes/web.php
Route::get('/', function () {
    return view('welcome', [
        'greeting' => 'Hello',
        'person' => request('person', 'World'), // クエリ文字列を読み取る。デフォルト値は 'World'
    ]);
});
```

これは Route::view の例と機能的には同一ですが、レンダリング前にリクエストを検査したりロジックを実行したりする柔軟性を提供します。

## クエリ文字列の読み取り

URL で変数を制御したい場合は、クエリ文字列を読み取り、それをビューに渡してください。
リクエストヘルパーは第二引数としてデフォルト値を受け付けます。これは便利です：

```php
// routes/web.php (inside closure)
$person = request('person', 'World'); // ?person=Sarah -> 'Sarah', otherwise 'World'
return view('welcome', ['greeting' => 'Hello', 'person' => $person]);
```

ここで /?person=Sarah をリクエストすると「Hello, Sarah.」が返されます。
クエリキーが存在しない場合、デフォルト値 ('World') が使われます。

## ブレードエスケープと XSS 対策

これは PHP の基本中の基本ですが、繰り返し強調しておきます：信頼できない生の入力をそのまま HTML に表示してはいけません。
Blade の二重中括弧はデフォルトでエスケープ処理を行います：

```html
<!-- 安全 -->
<p>{{ $person }}</p>
```

もし誰かが ?person=<script>alert('xss')</script> を渡しても、Blade はこれをエスケープするためスクリプトは実行されません。
これが Blade の最も便利な機能の一つです。

意図的に生の HTML を出力したい場合（ソースを信頼できる場合のみ実行してください）、unescaped ディレクティブを使用できます：

```html
<!-- 危険: ソースを信頼できる場合のみ -->
{!! $trustedHtml !!}
```

注意してください ―― これによりエスケープ処理が完全に無効化されます。

## 例: ルート + ビュー + クエリ文字列 + ブレード

以下に、すべてをまとめて確認できるコンパクトな例を示します：

```php
// routes/web.php
Route::get('/', function () {
    return view('welcome', [
        'greeting' => 'Hello',
        'person' => request('person', 'World'),
    ]);
});
```

```html
<!-- resources/views/welcome.blade.php -->
<h1>{{ $greeting }}, {{ $person }}.</h1>
```

/?person=Simon にアクセスすると「Hello, Simon.」と表示されます。
/ にアクセスすると「Hello, World.」と表示されます。

## Route::view とクロージャーの使い分け

ページが静的である場合やデータが単純でロジックの実行が不要な場合は Route::view を使用してください。
リクエストの検査、データベース検索、またはレンダリング前のその他の計算が必要な場合は、ルートクロージャー（またはコントローラー）を使用してください。

どちらも同じビューデータ契約を受け入れます。
これはキーがビュー変数となる連想配列です。

## ちょっとしたコツ

クエリキーのデフォルト値が必要な場合は、request() の2番目の引数としてデフォルト値を渡してください。
複数のビューで共通して利用可能な値が必要な場合は、ビューコンポーザーや、値を一元的に注入するベースコントローラーメソッドの使用を検討してください。
そしてもちろん、非常に正当な理由がない限り、Blade のエスケープ済み出力を優先してください。
