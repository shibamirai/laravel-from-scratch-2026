# ルーティング入門

私が Laravel の好む理由について話しましょう：
その API はまるで普通の言語のように感じられます。
何が起きているのかを理解するのにフレームワークの達人である必要はありません — コードがほぼ自らを説明しているかのようです。
その感覚を実際に体験しながら、基本を解説していきます：
ルーティング、ビュー、そして小さなアプリでこれら二つの要素がどう連携するかです。

## ルーティングは英語のように読める

新しい Laravel アプリでこのようなものを見かけたとき、事前知識がなくてもおそらく解析できるでしょう：

```php
// routes/web.php
Route::get('/', function () {
    return view('welcome');
});
```

一緒に声に出して言ってみましょう：「ホームページへのGETリクエストに応答し、`welcome` ビューを返すルートを登録する」
以上です。メソッド（`get`）は HTTP メソッド、最初の引数は URI、コールバックはその URI にアクセスした際に実行する処理です。

### ビューとはなにか

> ビュー (view) とは、ユーザーがウェブページ上で目にするテンプレート（HTML、CSS、場合によっては JavaScript）のことです。

言い換えれば、ビューとはブラウザ上で「表示」される対象です。
それはフルページ、部分ページ、あるいはコンポーネントである可能性がありますが、概念的には UI をレンダリングするテンプレートを指します。

## ウェルカムビューを見つけよう

デフォルトでは、`welcome` ビューはリソースディレクトリ内に配置されます。こちらをご覧ください：

```text
resources/views/welcome.blade.php
```

そのファイルを開くと、Laravel をインストールした際に初期状態で表示されるスプラッシュページが表示されます。

## Laravel Herd でアプリケーションを見る

Laravel Herd を使用している場合、ブラウザで Herd が公開しているフォルダ名（例：`test`）にアクセスするだけで、スプラッシュページが表示されます。
`welcome.blade.php` を編集し、テキストを変更してブラウザを更新すると、変更内容がリアルタイムで反映されます。

## ビューの編集 (簡単な例)

ビューの編集は次のような感じになります ―― コンテンツをシンプルな「Hello World」に置き換える：

```html
<!-- resources/views/welcome.blade.php -->
<h1>Hello World</h1>
```

保存して更新すれば、すぐに使えるようになります。

## ページのリンクとアバウトページの追加

フレームワークを学ぶ際、最初に抱く疑問の一つは「ページAからページBへどうリンクするか？」です。
`welcome.blade.php` にアンカーを追加しましょう：

```html
<a href="/about">About Us</a>
```

クリックすると `/about` へのアクセスを試みます。
`/about` へのルートを登録していない場合、Laravel は 404 を返します — まさに期待通りの動作です。
それでは修正しましょう。

### アバウトページの登録

`routes/web.php` を開き、ホームページのルートをコピーして、`/about` 用の新しいルートを登録してください。
注記: 最初のスラッシュを含めても省略しても構いません ―― **どちらか一方を選択し、一貫して使用してください**。

```php
Route::get('/about', function () {
    return view('about');
});
```

ビューの代わりにプレーンな文字列を（一時的に）返したい場合、次のようにできます：

```php
Route::get('/about', function () {
    return 'About Us';
});
```

素早いテストには便利ですが、通常はビューを返すことになります。

## アバウトページの作成

新しいビューファイルを作成します：

```text
resources/views/about.blade.php
```

好きなマークアップで埋めてください。例えば：

```html
<!-- resources/views/about.blade.php -->
<h1>About Us</h1>
<p>Welcome — this is a placeholder about page.</p>
<p><a href="/">Return Home</a></p>
```

ブラウザで `/about` を再読み込みすると、about ページが表示されるはずです。
ページ間で共通のマークアップが重複していることに気づくでしょう — 心配しないでください。
Blade レイアウトと部分テンプレートで、すぐにこの問題を解決します。

## 宿題：連絡先ページを追加する

さあ、小さなテストです。ローカルで実行してください：

1. `/contact` への GET リクエストを監視し、`contact` ビューを返すルートを登録します。例：

    ```php
    Route::get('/contact', function () {
        return view('contact');
    });
    ```

1. ビューファイル `resources/views/contact.blade.php` を作成し、現時点ではダミーコンテンツで埋めてください：

    ```html
    <!-- resources/views/contact.blade.php -->
    <h1>Contact</h1>
    <p>Send us a message (this is just dummy content for now).</p>
    ```

実際のフォームについては後で説明するので、今回はシンプルに済ませてください。
試してみて、結果を教えてください。
何か問題が起きたら、ルートとビューのコードを貼り付けてください。デバッグをお手伝いします。
