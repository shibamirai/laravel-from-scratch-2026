# アイキャッチ画像をストレージにアップロードする

アイデアにアイキャッチ画像を追加することを考えましょう。
登録／編集フォームにはシンプルなファイル入力欄、コントローラにはバリデーションと保存、アップロードしたファイルを公開する方法、そしてアイデアページやアイデアカードでそれらのサムネイル表示が必要です。
早速行います ― 順に変更点や引っかかりやすい箇所をお見せしながら進めます。

> multipart/form-data はフォームにブラウザからのファイルアップロードを許可するものです。

ファイル送信するときには忘れてはいけない１行です。

## フォームにファイル入力を追加する

まず、登録／編集フォームの詳細の下にファイル入力を追加します。
他と同じようにラベルとバリデーションエラーの UI を忘れないでください。

そのままモーダル／フォームに貼り付けできるシンプルな例です：

```html
<div class="mt-4">
  <label class="label" for="image">Featured Image</label>
  <input id="image" name="image" type="file" accept="image/*" class="input" />
  @error('image')
    <p class="text-red-500 text-sm mt-1">{{ $message }}</p>
  @enderror
</div>
```

そしてとても重要なのが ― フォームでファイルをエンコードさせます：

```html
<form method="POST" action="{{ route('ideas.store') }}" enctype="multipart/form-data">
  @csrf
  <!-- 各フィールド... -->
</form>
```

## アップロードのバリデーション（フォームリクエストもしくはコントローラ）

画像のバリデーションを追加しましょう。
フォームリクエストを使用している場合（推奨）は rules を更新して、未指定可、画像であること、サイズ制限（例：5MB = 5120KB）のルールを設定しましょう。


```php
// app/Http/Requests/StoreIdeaRequest.php (excerpt)
public function rules()
{
    return [
        'title' => ['required', 'string', 'max:255'],
        'description' => ['required', 'string', 'min:4'],
        'image' => ['nullable', 'image', 'max:5120'], // 5MB
    ];
}
```

バリデーションに失敗すると、用意していたエラーの UI にメッセージが表示されるでしょう。

## コントローラでアップロードしたファイルを保存する

Laravel ではファイル保存が驚くほど簡単です。
アイデアを作成した後、アップロードした画像を保存してそのパスをアイデアのレコードにセットします。

こちらが store アクションでの処理を簡略化したものです：

```php
// app/Http/Controllers/IdeaController.php (excerpt)
public function store(\App\Http\Requests\StoreIdeaRequest $request)
{
    $idea = auth()->user()->ideas()->create($request->only('title', 'description', 'status'));

    if ($request->hasFile('image')) {
        // storage/app/public/ideas に保存し、パス "ideas/filename.jpg" を返す
        $path = $request->file('image')->store('ideas', 'public');

        // 相対パスをアイデアに保存（'image' カラムがあることを前提しています）
        $idea->update(['image' => $path]);
    }

    // リダイレクトするか通知するか...
    return redirect()->route('ideas.show', $idea);
}
```

私は `public` ディスクに保存するのが好みです。
そうすると public storage のシンボリックリンク（下記参照）でファイルを公開しやすいからです。
保存先のパスは `ideas/ユニークな名前.jpg` のような感じになります。

アップロードされたファイルの情報を確認したければ、こうしてみましょう：

```php
dd($request->all(), $request->file('image'));
```

...でも確認後には削除してください。

## アップロードされたファイルを公開する

デフォルトではアップロードされたファイルはストレージに入れられ Web からアクセスできません。
artisan コマンドを実行して public/storage から storage/app/public にシンボリックリンクを張ってください：

```bash
php artisan storage:link
```

こうすると `public/storage` が `storage/app/public` を指すようになります。
これで `public` ディスクに保存された画像は `/storage/...` でアクセスできます。

## ビューで画像を表示

アイキャッチ画像を表示（もしあれば）するには、画像があるかを確認してから asset ヘルパーか Storage::url() を使用します：

```html
{{-- resources/views/ideas/show.blade.php --}}
@if ($idea->image)
  <div class="mb-4 overflow-hidden rounded-t-lg">
    <img
      src="{{ asset('storage/' . $idea->image) }}"
      alt="{{ $idea->title }} featured image"
      class="w-full h-48 object-cover"
    >
  </div>
@endif

<h1 class="text-2xl font-bold">{{ $idea->title }}</h1>
```

もしくは、お好みで Storage::url($idea->image) とすることもできます。

```php
<img src="{{ Storage::url($idea->image) }}" ...>
```

Blade がエスケープ処理を行うので、ここで値をそのまま表示しても安全です。

## アイデアカードにサムネイルを表示する

インデックスカードにサムネイルをつけたければ、同じ画像ブロックをカードコンポーネントの上につけ、余白を調節してカードの端まで広がるようにします。例えば：

```html
{{-- resources/views/components/idea-card.blade.php (excerpt) --}}
@if ($idea->image)
  <div class="-mx-4 rounded-t-lg overflow-hidden">
    <img src="{{ asset('storage/' . $idea->image) }}" class="w-full h-40 object-cover" alt="">
  </div>
@endif

<div class="p-4">
  <h2>{{ $idea->title }}</h2>
  <p>{{ $idea->description }}</p>
  <!-- ... -->
</div>
```

画像ラッパーの水平マージンに負の値をセットすることでサムネイルをカードの端まで引き延ばしつつ、下側には余白を残しています。

## スタイリングのコツ

`object-cover` を使って画像を歪ませずにうまくボックスを埋めるようにし、サムネイルの大きさが同じになるように高さを固定します。
コンテナ内の画像を `overflow-hidden` と `rounded-t-lg` でラップして、上部の角を丸くします。

## スキーマの確認

アイテムテーブルの画像パスのカラムがこんな風に Null 可になっているか確認しておきましょう：

```php
// database/migrations/xxxx_xx_xx_create_ideas_table.php (抜粋)
$table->string('image')->nullable();
```

## 次に考慮すべきこと

おそらく次に考えるのは、編集モーダルでアイキャッチ画像の削除か更新をサポートすることでしょう。
それにはファイルを削除 (Storage::disk('public')->delete($idea->image)) してカラムをクリアする必要があります。
また、コントローラにたくさんのロジックが積みあがっていることに気づくでしょう ― アイデアの作成に複数のステップ（データベースへの保存、ファイルの保存、手順の追加、通知の発出）が含まれるのだとしたら、コードを重複させずにロジックを再利用できるように専用のアクション／サービスクラスを用意することを考えましょう。
