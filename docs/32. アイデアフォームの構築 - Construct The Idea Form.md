# [アイデアフォームの構築](https://laracasts.com/series/laravel-from-scratch-2026/episodes/32)

アイデア登録フォームの構築について話しましょう ― モーダルな UI、保存するためのルートとコントローラ、Alpine.js を利用した状態セレクタ、これらを動作させるために必要なあれこれ（CSRF、メソッド偽装）についてです。
皆さんがついてこれるよう順に実装していきます。

## 保存のためのルートとコントローラアクションの追加

まず、フォームには送信先が必要です。
ideas に POST ルートを追加して IdeaController の store アクションに紐づけ、auth で保護しました。

```php
// routes/web.php
Route::post('/ideas', [App\Http\Controllers\IdeaController::class, 'store'])
    ->name('ideas.store')
    ->middleware('auth');
```

コントローラでは実際にアイデアを保存します；とりあえず store アクションでリクエストを受け付け、後で何かしらの保存用ヘルパーを呼び出しましょう。

```php
// app/Http/Controllers/IdeaController.php (excerpt)
public function store(\App\Http\Requests\StoreIdeaRequest $request)
{
    // アイデアの保存 ー 以下に実装
}
```

フォームリクエスト (StoreIdeaRequest) を使って、コントローラの実行前に受信データのバリデーションを認可を行います。

## モーダルなフォームのマークアップ

登録フォームはモーダルコンポーネント内に配置します。
フォームは store ルートへ POST 送信しで CSRF 保護を含めなければいけません。
モーダルの中身はこのようになります：

```html
<form method="POST" action="{{ route('ideas.store') }}" enctype="multipart/form-data">
  @csrf

  <!-- タイトル欄 -->
  <x-field label="Title" name="title" placeholder="Enter an idea title" required />

  <!-- 詳細欄 (textarea) -->
  <x-field label="Description" name="description" type="textarea" placeholder="Describe your idea" />

  <!-- 状態ボタンと hidden データは Alpine で操作する（下記参照） -->

  <!-- フッターのボタン欄 -->
  <div class="flex gap-2">
    <button type="button" @click="$dispatch('close-modal')">Cancel</button>
    <button type="submit" class="btn-primary">Create</button>
  </div>
</form>
```

補足：

- 後でファイルアップロード（画像）を追加するために `enctype="multipart/form-data"` があります。
- 上記の `x-field` はラベルと input/textarea とエラーをラップしたシンプルな Blade コンポーネントです。`type="textarea"` が要求されたら input を textarea に置き換えます。
- キャンセルボタンで `close-modal` イベントを発生させ、それをモーダルが受け取ります（それによって非表示にして入力もリセットします）。

## フォームリクエストによるバリデーション

store データをバリデーションするためにフォームリクエスト (StoreIdeaRequest) を使いました。
今は `title` だけを必須にしています；その他は未入力可：

```php
// app/Http/Requests/StoreIdeaRequest.php (excerpt)
public function authorize()
{
    return true; // 今はすべて許可（または認証チェックを行う）
}

public function rules()
{
    return [
        'title' => ['required', 'string', 'max:255'],
        'description' => ['nullable', 'string'],
        'status' => ['required', 'in:pending,in_progress,completed'],
        // 後で追加予定: 'image' => ['nullable','image','max:5120']
    ];
}
```

フォームリクエストはコントローラでタイプヒントしていれば自動で実行されるので、`$request->validate()` を呼ぶ必要はありません。

## Alpine.js による状態セレクタ

３つのボタン（保留、実施中、完了）を用意して、１つをクリックするとそれが選択されたことが分かるような見た目にして、選択された状態が hidden データとしてフォームで送信されるようにしたいと思います。
それには Alpine がぴったりです。

- フォーム（またはその親要素）を `x-data` を持つ Alipine のルートとする。
- 状態の各ボタンはクリックされると `selected` にその状態をセットする。
- hidden の `<input name="status">` と `selected` をバインドし、フォームと一緒に送信されるようにする

このようになります：

```html
<form x-data="{ selected: 'pending' }" method="POST" action="{{ route('ideas.store') }}">
  @csrf

  <label class="label">Status</label>

  <div class="flex gap-2">
    <button
      type="button"
      :class="selected === 'pending' ? 'btn-primary' : 'btn-outlined'"
      @click="selected = 'pending'"
    >
      Pending
    </button>

    <button
      type="button"
      :class="selected === 'in_progress' ? 'btn-primary' : 'btn-outlined'"
      @click="selected = 'in_progress'"
    >
      In Progress
    </button>

    <button
      type="button"
      :class="selected === 'completed' ? 'btn-primary' : 'btn-outlined'"
      @click="selected = 'completed'"
    >
      Completed
    </button>
  </div>

  <!-- 送信される hidden データ -->
  <input type="hidden" name="status" :value="selected" />
</form>
```

こうすると、送信する値はサーバが期待する簡単なフォームフィールドにしたままで、クライアント側で素晴らしいユーザ体験を提供します。

## メソッド偽装 (Patch/Put/Delete)

ブラウザはそのままでは GET と POST しかサポートしません。
RESTful な更新や削除を行いたい場合、Laravel ではメソッド偽装を使います：POST を使いますが、中に hidden で `_method` フィールドを含めて送信します。

> メソッド偽装：ブラウザからは POST で送信しつつ、hidden フィールドを使って（`@method('PATCH')` や `@method('DELETE')`）本当に利用したい HTTP メソッドを Laravel に伝える。

アイデアの更新を行う編集フォームの例：

```html
<form method="POST" action="{{ route('ideas.update', $idea) }}">
  @csrf
  @method('PATCH') <!-- ブラウザの POST を通じて PATCH を知らせる -->
  <textarea name="description">{{ $idea->description }}</textarea>
  <button type="submit">Update</button>
</form>
```

これに対応するルート定義：

```php
Route::patch('/ideas/{idea}', [IdeaController::class, 'update'])->name('ideas.update');
```

削除には別のフォームを用意（他のフォームの中にネストさせない）し `DELETE` に偽装する：

```html
<form method="POST" action="{{ route('ideas.destroy', $idea) }}" id="delete-idea-{{ $idea->id }}">
  @csrf
  @method('DELETE')
  <button type="submit">Delete</button>
</form>
```

ブラウザはネストしたフォームを扱えません。
もし削除ボタンを編集フォームのそばに表示させる場合は、削除フォームを別に用意して削除ボタンからそのフォームを指定しましょう（または単に編集フォームの後に削除フォームを配置する）。

## モーダルの UX：閉じるボタンとクリックアウト

クリックアウトでモーダルを閉じることができる場合でも、アクセシビリティの観点から明示的に閉じるボタンを用意したほうがいいです。
ボタンには `Close modal` といった `arai-label` をつけ、`@click="$dispatch('close-modal')"` としてモーダルにイベントを送り、閉じられるようにしましょう。

モーダルコンポーネントでは `open-modal` と `close-modal` の両方のイベントをウィンドウレベルで受信するので、このコンポーネントを含んだ様々なページで同じイベントに反応できるでしょう。

## 登録処理（正常系）

手っ取り早くシンプルな登録処理です（後でいろいろ処理を追加します ― 画像、手順、通知など）：

```php
// IdeaController@store
public function store(\App\Http\Requests\StoreIdeaRequest $request)
{
    $attributes = $request->validated();

    // ログインユーザに紐づける
    $idea = auth()->user()->ideas()->create($attributes);

    return redirect()->route('ideas.index')->with('success', 'Idea created.');
}
```

`auth()->user()->ideas()` という整理された形でリレーションを使って、正しく関連付けます。

## 微調整

コーディング中にいくつかちょっとした UX の改善を行いました。

- モーダルに適当な max-width と max-height (80vh) をセットし、大きなフォームがビューポートを超えないように内部でスクロールできるようにしました。
- 見出しと閉じるアイコンを追加し、flex で配置（見出しを左、閉じるを右）しました。
- モーダルが見やすくなるように、フォームフィールドに余白（`space-y` や `mt-#`）をつけました。
- 登録ボタンを `type="submit"` に、キャンセルボタンを `type="button"` にしてキャンセル時に送信されないようにしました。

## 一覧の並び（新しい順）

デフォルトではクエリは古いもの順で全レコードを返します。
新しく登録されたアイデアから順に表示されるようにクエリを新しいもの順にしました。

```php
$ideas = Idea::latest()->get(); // created_at の降順で並び替え
```

フィードや一覧表示での UX のコツです。

## まとめ

これでモーダル内部にアイデアの登録フローができました：

- イベントと Alpine によってモーダルの表示を切り替えます。
- フォームは `auth` で保護された store ルートに POST します。
- 状態セレクタは Alpine によって hidden データと紐づけられた操作可能な UI です。
- PATCH/DELETE が必要なときはメソッド偽装（`@method`）を使いましょう。
- フォームリクエストでバリデーションを行い、作成したアイデアは `auth()->user()->ideas()->create()` でログインユーザに紐づけましょう。

必要であれば、自身のプロジェクトにそのまま追加できる モーダルの Blade コンポーネント、コントローラの完全な store/update メソッド、状態選択を行う Alpine スクリプト、をお渡しできます。
