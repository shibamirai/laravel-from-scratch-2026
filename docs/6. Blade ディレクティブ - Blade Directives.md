# [Blade ディレクティブ](https://laracasts.com/series/laravel-from-scratch-2026/episodes/6)

Blade ディレクティブについて話しましょう。
これは Laravel でのビューレンダリングを、生の PHP よりもずっと快適にする、素早く便利なショートカットです。
私がよく使う最も一般的なものを紹介します：ダンプとデバッグ、条件分岐、ループ、そしていくつかの認証ヘルパーです。
これらは入力の手間を大幅に省き、ビューの可読性を高めるものです。

## Blade ディレクティブの概要

> Blade は PHP の上に構築されたテンプレートエンジンです。

Blade で記述した内容は最終的にプレーンな PHP にコンパイルされるため、裏側ではすべてシンプルな PHP コードとなります。
つまりディレクティブは単なる構文糖衣であり、記述が楽で読みやすいという利点があるだけです。

## ダンプとデバッグ

ビュー内で変数を検査する必要がある場合、生の PHP を使用できます：

```php
<?php
var_dump($tasks);
```

あるいは昔ながらのダイダンプが使えます：

```php
<?php
dd($tasks); // var_dump + 実行を停止
```

しかし Blade では、より洗練された構文が得られます。
@dump ディレクティブを使用すると、実行を停止せずに変数を表示できます:

```php
@dump($tasks)
<h2>Tasks</h2>
```

ダンプして終了したいときは @dd を使いましょう（コミュニティは **DD** をこよなく愛しているのでどこでも見かけるでしょう）：

```php
@dd($tasks)
<!-- 実行を停止するので以下は表示されません -->
<h2>Won't show</h2>
```

どちらも開発中に便利で、PHP タグを何度も開いたり閉じたりするよりもずっと良いでしょう。

## 条件文

Blade は、大量の角括弧の代わりに、条件分岐のための読みやすいディレクティブを提供します。

従来のPHP：

```php
<?php if (count($tasks)): ?>
    <p>We have some tasks (<?php echo count($tasks); ?>).</p>
<?php endif; ?>
```

Bladeでは：

```php
@if (count($tasks))
    <p>We have some tasks ({{ count($tasks) }}).</p>
@endif
```

@unless を使えば、より自然な "if not" という条件表現もできます:

```php
@unless (count($tasks))
    <p>There are no current tasks.</p>
@endunless
```

これらはすべて PHP の条件チェックと同等ですが、はるかに見やすいです。

## ループと空の状態

ループ処理は単純明快です。
タスクを反復処理しリスト項目をレンダリングする方法は以下の通りです：

```php
<ul>
@foreach ($tasks as $task)
    <li>{{ $task }}</li>
@endforeach
</ul>
```

コレクションが空の場合があるなら、@forelse と @empty ブロックを使用してください。
これは「アイテムを表示するか、アイテムがないメッセージを表示する」という一般的なケースに最適です:

```php
<ul>
@forelse ($tasks as $task)
    <li>{{ $task }}</li>
@empty
    <li>There are no active tasks.</li>
@endforelse
</ul>
```

これは従来の if チェックと foreach の組み合わせに代わるもので、はるかに簡潔です。

## 認可と便利なディレクティブ

Blade には認証と認可チェックのためのディレクティブも含まれています。
これらは読みやすく、ビューロジックを整理した状態に保ちます。

認証済みユーザーを確認する：

```php
@auth
    <p>Welcome back, {{ auth()->user()->name }}.</p>
@endauth
```

権限/ポリシーの確認（例：現在のユーザーが投稿を編集できるかどうか）：

```php
@can('update', $post)
    <a href="{{ route('posts.edit', $post) }}">Edit</a>
@endcan
```

これらは内部では通常の認証／認可呼び出しにコンパイルされますが、Blade テンプレートの表現力を維持します。

## 最後に

ブレードディレクティブは最終的に PHP となる便利な機能です。
そのため、通常の PHP で学んだ知識は全てそのまま適用できます。
迅速なデバッグには @dump / @dd から始め、読みやすいロジックには @if / @unless と @foreach / @forelse を、ユーザー状態や権限を扱う際には @auth / @guest / @can を活用しましょう。
使い込むうちに、最も頻繁に使うディレクティブがいくつか見つかるでしょう。
それらはビューの記述と読解の両方を加速させてくれます。
