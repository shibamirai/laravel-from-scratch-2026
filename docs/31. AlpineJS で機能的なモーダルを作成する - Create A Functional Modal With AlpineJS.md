# [AlpineJS で機能的なモーダルを作成する](https://laracasts.com/series/laravel-from-scratch-2026/episodes/31)

登録／編集モーダルの構築について話しましょう ― クリックするとモーダルが開くカードや、Alpine からイベントを出す方法、再利用可能で使いやすいモーダルコンポーネントの構築方法についてです。
実際に動かせる実践的なもの（となぜそうするかの理由）をお見せしましょう。

## クリックするとモーダルが開くカードの作成

インデックスページで大きなボタンのように見えるシンプルなカードの作成から始めましょう：ユーザがクリックするとモーダルが開くボタンです。
用途によってアンカーにもボタンにもできるフレキシブルな要素にしたいと思います。

ボタンとしてレンダリングしたカードの例です（Tailwind のクラスを付与した Blade / HTML）：

```html
<!-- resources/views/ideas/index.blade.php -->
<x-card as="button"
        type="button"
        class="w-full cursor-pointer h-32 text-left p-6"
        @click="$dispatch('open-modal', { name: 'create-idea' })">
  <p class="text-lg font-medium">What's the idea?</p>
</x-card>
```

２点補足します。
まず、明示的に `type="button"` とセットして、偶然近くのフォームを送信してしまわないようにしています。
次に、テキストで適切な要素を探すのではなく、名前付きイベントを発生させています ― そうすることでアクションが分かりやすく、テストしやすくなります。

## Alpine からブラウザイベントを発生させる

Alpine の `$dispatch()` を使うと、他のコンポーネントが受け取れるブラウザイベントを簡単に発生させることができます。
私はいつもそれに追加情報を含め、名前によって１つのイベントタイプで多くのモーダルを開けるようにしています。

いつもこのようにします：

```html
<!-- dispatching the event -->
<button @click="$dispatch('open-modal', { name: 'create-idea' })">
  Create Idea
</button>
```

コンソールで確認してみましょう：

```javascript
window.addEventListener('open-modal', (e) => {
  console.log('open-modal detail:', e.detail);
});
```

`datail` オブジェクトは先ほど渡した追加情報です ― このケースでは `{ name: 'create-idea' }` です。
これを使えば、`event.detail.name` を比較することで異なるモーダルを開くことができます。

## モーダルを作成し、イベントを受け取る

モーダルはそれが必要なページにあらかじめ配置し（またはコンポーネントとして含め）ておきますが、デフォルトでは表示しません。
モーダルはウィンドウの open イベントを受け取り、イベント名がマッチしたときに表示を切り替えます。

Alpine を使った簡単で便利なモーダルコンポーネントのパターンです：

```html
<!-- resources/views/components/modal.blade.php -->
@props(['name' => 'modal', 'title' => 'Modal Title'])

<div x-data="{ show: false, name: @js($name) }"
     x-show="show"
     x-cloak
     x-on:open-modal.window="if ($event.detail?.name === name) show = true"
     x-on:keydown.escape.window="show = false"
     class="fixed inset-0 z-50 flex items-center justify-center">
  <div class="fixed inset-0 bg-black/50" @click="show = false" aria-hidden="true"></div>

  <div role="dialog"
       :aria-hidden="!show"
       :aria-labelledby="'modal-title-'+name"
       tabindex="-1"
       class="bg-white rounded-lg shadow-lg max-w-2xl w-full p-6 transform transition">
    <header>
      <h2 id="modal-title-{{ $name }}" class="text-lg font-semibold">{{ $title }}</h2>
    </header>

    <main class="mt-4">
      {{ $slot }}
    </main>
  </div>
</div>
```

いくつかの大事なこと：

- `x-cloak`（とその CSS ルール）によって、Alpine の起動前にモーダルがちらっと見えてしまわないように隠しています。
- `x-on:open-modal.window` でイベントをグローバルに受け取っています。`event.detail.name` をチェックすることでこのコンポーネントが他と共存できるようにしています。
- `x-on:keydown.escape.window` によってエスケープキー（esc）で閉じることができます。
- `role="dialog"`, `aria-labelledby`, `aria-hidden` によってスクリーンリーダーがモーダルを判別できるようにしています。

ビューではこのように使用します：

```html
<x-modal name="create-idea" title="New Idea">
  <!-- ここにフォームを配置します -->
</x-modal>
```

## スムーズな変化

Alpine の `x-transition` によって開始／終了の変化をつけることができます。
洗練された見た目になるよう変化は素早くほんの少しだけになるようにしています。

```html
<div x-transition:enter="duration-200 ease-out"
     x-transition:enter-start="opacity-0 translate-y-4"
     x-transition:enter-end="opacity-100 translate-y-0"
     x-transition:leave="duration-150 ease-in"
     x-transition:leave-start="opacity-100 translate-y-0"
     x-transition:leave-end="opacity-0 translate-y-2">
  <!-- モーダルの内容 -->
</div>
```

これにより、モーダルが現れるときと消えるときに穏やかなスライドとフェードがつきます。

## クリックとキーボード操作への対応

クリックすればモーダルを閉じる領域を用意します（モーダルの背後に追加しました）。
また以前ご紹介したように `keydown.escape.window` のリスナーを使ってエスケープキーでも閉じれるようにしましょう。
誰でも使えるようにするには、フォーカス移動（モーダルが開いたらその中にフォーカスを移動し、各要素にフォーカスを切り替える）もできるようにすべきでしょう。
様々なライブラリで実現可能です；Alpine で簡単に実装したければ、フォーカスを取得する小さなスクリプトがありますし、もしくはコンポーネントに簡単なフォーカスマネージャを実装することもできます。

## アクセシビリティに関して考慮すべきこと

- role="dialog" と aria-labelledby は重要です。
- ダイアログが非表示になっているときは `aria-hidden` を使いましょう。
- Alpine 起動前に一瞬だけ表示されることがないよう `x-cloak` を使いましょう。
- 分かりやすくなるように、モーダルのラベルに `modal-title-create-idea` といったユニークな ID をつけましょう。
- リリースまでにキーボード操作に対応（エスケープキー、タブによるフォーカス移動）させましょう。

## 再利用可能なコンポーネントに取り出す

モーダルを Blade コンポーネントに取り出して、同じことを何度も書かないで済むようにし、インタフェースをシンプルにしましょう。

```html
<x-modal name="create-idea" title="New Idea">
  <form method="POST" action="/ideas">
    @csrf
    <x-forms.field name="title" label="Title" />
    <x-forms.field name="description" label="Description" type="textarea" />
    <div class="mt-4 flex justify-end gap-2">
      <button type="button" class="btn-ghost" @click="$dispatch('close-modal', { name: 'create-idea' })">Cancel</button>
      <button type="submit" class="btn btn-primary">Create</button>
    </div>
  </form>
</x-modal>
```

コンポーネントを拡張して、幅や閉じるコールバック、またはフォーカストラップに含めるかどうかなどを prop として受け取るようにもできます。

## テストとデバッグのコツ

- （Pest と Playwrightを使った）ブラウザテストの作成中は、モーダルの状態を確認できるよう `pause()` や `debug()` を追加しましょう。
- Alpine の初期化前にモーダルが一瞬現れるようなら `x-cloak` という CSS ルールを追加しましょう。
  ```css
  [x-cloak] { display: none !important; }
  ```
- テストで対象となる要素を確実に選択できるように、対象の要素に `data-test` 属性をつけましょう。
- サーバでレンダリングしたリンクからのイベントでモーダルを表示する場合は、そのリスナーが確実に存在するように、必ずモーダルコンポーネントを同じページ（もしくはグローバル）に含めるようにしましょう。
