# ここから次のステップ

このプロジェクトを終えた今、次に何をしていくのかについて話しましょう。
ここまでたどり着いたみなさん ― 本当にありがとう。
このコースを準備するのには、見た目よりももっと多くの時間がかかっています（企画して、録画して、編集して ― 10倍はかかかりました）。
ここまで一緒に進めていただいたことを心から感謝しています。

## このプロジェクトをどうしていくか

このリポジトリは完全なオープンソースなので、自由にフォークして、改造して、またはバラバラにしてください ― 全く構いません。
もっと練習したければ、実際に人が欲しがる機能を構築してみましょう。
例えばチームを導入してみるのはいい考えです：
ジョンとジェーンを同じチームにしてアイデアを共有して共同作業できるようにしてみましょう。
ただし、認可ルールを追加してジョンが勝手にジェーンのアイデアを更新／削除できないようにする必要があります。
こういった実際によくある要求はとてもいい練習になります。

## 学習を続けよう

学習はここで終わりではありません ― 正直にいうと、決して終わることはないでしょう。
そういうものです。
興味あることを見つけてそれを深めていってください。

> Livewire を使うと、JavaScript を使わずに PHP だけでインタラクティブなアプリを構築できます。

Livewire は動的で刺激的なフレームワークです。
PHP プログラミングが好きで、インタラクティブ性に対する全く新しいアプローチを求めている方に最適です。

> Inertia.js を使うと、これまでのサーバ側でのルーティングと認可を使いながら SPA の利点を味わえます。

フロントエンドには Vue.js や React を使いつつ、Laravel のルーティングやバックエンドのロジックはそのまま使いたいという人にとって、Inertia.js はその間を受け持つ素晴らしい架け橋となってくれます。
Laracasts で私はいつも Inertia を使っているので、私のコースをフォローすればどういうものかを見ていただけます。

もし JavaScript でコードを書くのが好きであれば、Vue.js や React を勉強して Laravel バックエンドとともに使う方法を学びましょう。
どちらも Inertia とペアで扱うと、フロントエンドとバックエンドの両方からアプローチするのに最適です。

## 挑戦してみることの例

実践的な学習に適した具体的な機能アイデアをいくつかご紹介します：チームサポートと権限管理の追加、アイデアのための画像アップロード・保存機能の実装、すべての変更を追跡するアクティビティログの構築、あるいは冒険心があるなら WebSocket を使ったリアルタイム共同編集機能の追加などです。
これらの各機能は異なるスタックの部分に触れるため、それぞれが新しい学びをもたらします。
