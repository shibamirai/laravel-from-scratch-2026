# [アイデアカード](https://laracasts.com/series/laravel-from-scratch-2026/episodes/28)

以前実施したモーダルとフォームの作業を、実際のアイデアのリスト化作業に適用しましょう ― ホームページを一時的に /ideas にルーティングし、インデックスビューを作成し、アイデアカードをスタイリングし、再利用可能なカードコンポーネントとして取り出し、現在状態を示すピルバッジを各カードに用意します。
変更内容とその理由を説明しながら、そのままプロジェクトで再利用できるコードを提供します。

## 簡単なコード品質テスト

新たにコードを書き始める前に、CodeRabbit を動かして何か問題が発生していないか確認してみました。
するといくつか役立つ情報がありました：パスワードのハッシュ化を提案し（実際には前にキャストで実施しています）、ログアウト時にセッショントークンの無効化と再生成を行うよう指摘し（いい考えです）、Request 型と request() ヘルパーのどちらを使うかが統一されていないことを知らせ（一貫するよう片方を選びました）、フォーム内で閉じられてない div に目印をつけ、nav で alt 属性が重複しているのを見つけました。
これらは見逃しやすい些細なことですが、早めに修正しておくに越したことはありません ― 前のコミットですべて修正しました。

## ルーティングとインデックスアクション

マーケティングページが整備されるまで、一時的にホームページを /ideas にリダイレクトすることにしました。
アイデアを一覧表示するルートを追加し、auth ミドルウェアで保護します。

routes/web.php に以下を追加します：

```php
use App\Http\Controllers\IdeaController;

Route::get('/', fn () => redirect('/ideas'));

Route::get('/ideas', [IdeaController::class, 'index'])
    ->middleware('auth')
    ->name('ideas.index');

Route::get('/ideas/{idea}', [IdeaController::class, 'show'])
    ->middleware('auth')
    ->name('ideas.show');
```

引っかかりやすい箇所：ゲストがホームページにアクセスすると /ideas にリダイレクトされますが、そこには認証が必要なため Laravel はログインと名付けられたルートにリダイレクトさせます。
リダイレクトが機能するように、ログインと名付けられたルート（ルート名が login）を用意しておくことを忘れないでください。

## インデックスコントローラ：認証されたユーザのアイデアを取得する

全件取得してからフィルタリングするのではなく、認証ユーザからスコープクエリを使用します。
コントローラではシンプルにこのように記述します：

```php
<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class IdeaController extends Controller
{
    public function index(Request $request)
    {
        $ideas = $request->user()->ideas()->latest()->get();

        return view('ideas.index', compact('ideas'));
    }

    public function show($id)
    {
        // アイデア詳細ページのスタブ
    }
}
```

ページネーションが必要になったらあとで get() を paginame() に置き換えます。

## アイデアビューの作成

resources/views/ideas/index.blade.php を作成し、まずは簡単なヘッダとアイデアの繰り返しだけを実装します。
作業中にいくつかの異なる方法でファイルを作成します（artisan、エディタのショートカット）― お好きな方法をお選びください。

Blade ビューのコアの部分です：レスポンシブなグリッドにより小さな画面では１カラム、大きな画面では２カラムで表示します。

```html
<div class="max-w-6xl mx-auto px-4 py-8">
  <header class="text-center mb-8">
    <h1 class="text-2xl font-bold">All Ideas</h1>
    <p class="text-sm text-muted-foreground">Your ideas, at a glance</p>
  </header>

  <div class="grid gap-6 lg:grid-cols-2">
    @forelse($ideas as $idea)
      <x-card>
        <h3 class="text-lg font-semibold text-foreground">{{ $idea->title }}</h3>
        <div class="mt-2 text-sm prose">
          {{ $idea->description }}
        </div>
        <div class="mt-4 text-xs text-muted-foreground">
          {{ $idea->created_at->diffForHumans() }}
        </div>
      </x-card>
    @empty
      <p>No ideas at this time.</p>
    @endforelse
  </div>
</div>
```

注意点：タイムスタンプを分かりやすく表示するために Carbon の diffForHumans() を使用しました。
また詳細は、後でマークダウンも表示できるようにするため、prose スタイルのブロックで表示しています。

## ダミーのアイデアのシーディング (Tinker)

グリッドのテストのために、ユーザ ID に自分を設定したアイデアをファクトリでいくつか作成しました。
Tinker でこのようにします：

```bash
php artisan tinker
>>> App\Models\Idea::factory()->count(3)->create(['user_id' => 8]);
```

user_id はデータベースに登録されている実際のユーザに合わせてください。
インデックスページをリフレッシュするとダミーの各アイテムのタイトルが表示されます。

## 再利用可能なカードコンポーネントの取り出し

同じカードのマークアップを繰り返し使用していたので、それを Blade コンポーネントとして取り出してビューをきれいで再利用可能な状態に保ちます。

resources/views/components/card.blade.php：

```php
@props(['class' => ''])

<div {{ $attributes->merge(['class' => 'rounded-lg border border-border bg-card p-4 text-sm ' . $class]) }}>
  {{ $slot }}
</div>
```

使い方は単純明快です：インデックス内のカードのマークアップをこのように置き換えます：

```html
<x-card>
  <!-- カードコンポーネント -->
</x-card>
```

これでインデックスビューは再利用可能な状態に保たれ、カードのスタイルを１か所でまとめて整えることができます。

## カードをクリック可能にする（アイデア詳細ページへのリンク）

カードをクリック可能にしてユーザがアイデアの詳細ページに遷移できるようにします。
その主な使い方に合わせて、今はカードをアンカータグにします。

インデックスビューで以下のようにします：

```html
<x-card as="a" href="{{ route('ideas.show', $idea) }}" class="block">
  <h3>...</h3>
  ...
</x-card>
```

内容によってカードをクリック不可にする必要が出てきた場合は、a タグを prop で受け取るようにしてコンポーネントが a か div のどちらでレンダーするか選択できるようにできます。

## 状態ラベルコンポーネント

アイデアの状態によって色が変化するようなピルバッジを用意したいと思います。
コンポーネントとして取り出せば、色の制御を一か所で行えます。

resources/views/components/idea/status-label.blade.php：

```php
@props(['status' => 'pending'])

@php
  $base = 'inline-block rounded-full px-3 py-1 text-xs font-medium';
  $classes = $base;

  if ($status === 'pending') {
      $classes .= ' bg-yellow-50 text-yellow-600 border border-yellow-100';
  } elseif ($status === 'in_progress') {
      $classes .= ' bg-blue-50 text-blue-600 border border-blue-100';
  } elseif ($status === 'completed') {
      $classes .= ' bg-green-50 text-green-600 border border-green-100';
  }
@endphp

<span {{ $attributes->merge(['class' => $classes]) }}>
  {{ \Str::headline($status) }}
</span>
```

カードでこのように使います：

```html
<x-idea.status-label :status="$idea->status" class="mt-1" />
```

保留、実行中、完了という状態に合わせて色が変化するピルバッジの完成です。
スタイリングのロジックが１か所に集まっているので、デザインを変えるのは簡単です。

## スタイリングのコツ

ラッパーテキストの色調を抑えることでカードのヘッドラインを際立たせ、状態ピルバッジの上のスペースを調整しました。
上マージンを加えたりカードの背景をセットしたりといった小さな調整が UI に一貫性を持たせます。
クエリーに `latest()` を加えて新しいアイデアが上に表示されるようにもしています。
