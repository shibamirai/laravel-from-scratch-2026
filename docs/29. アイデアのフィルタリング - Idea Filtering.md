# [アイデアのフィルタリング](https://laracasts.com/series/laravel-from-scratch-2026/episodes/29)

アイデアのフィルタリングについて話しましょう ― 完了済みのアイデアだけ表示したり、保留中のものだけ、などです。
実際に使えるコンパクトなやり方をお見せします：Eloquent の when を使った条件付きクエリ、groupBy による状態別の個数、すべての状態（０個であっても）が表示されること、それからロジックをモデルに移してコントローラやビューがきちんとした状態に保つ方法です。

## When を使った条件付きクエリ

> Eloquent の when メソッドは条件に応じてクエリに制約をつけます ― 第一引数が true であればコールバックを実行し、クエリビルダと引数の値を受け取ります。

これはコントローラのコードをきれいに保つ糖衣構文です。
意味するところはこうです：もしリクエストに status というクエリパラメータが含まれていれば、それを where('status', $status) 句に適用します；含まれていなければすべてのアイデアを返します。

コントローラのコード例：

```php
// app/Http/Controllers/IdeaController.php
use App\Models\Idea;

public function index()
{
    $status = request('status'); // 例 ?status=pending

    $ideas = auth()->user()->ideas()
        ->when($status, function ($query, $status) {
            $query->where('status', $status);
        })
        ->get();

    // 以下で $statusCounts を用意しビューに渡す
    $statusCounts = Idea::statusCounts(auth()->user());

    return view('ideas.index', compact('ideas', 'statusCounts'));
}
```

`status` がなければ when のコールバックはスキップされそのユーザのすべてのアイデアが返されます。

## Group By を使って状態ごとのアイデアの数を数える

フィルターの隣に個数を表示（例：Pending (4)）するために、グループ別のカウントを行います。
このような SQL です：

```sql
SELECT status, COUNT(*) AS count
FROM ideas
GROUP BY status;
```

Eloquent のクエリビルダで書くと（連想配列で個数を返します）：

```php
$counts = Idea::selectRaw('status, count(*) as count')
    ->when($user, fn($q) => $q->where('user_id', $user->id))
    ->groupBy('status')
    ->pluck('count', 'status'); // -> ['pending' => 4, 'in_progress' => 1, ...]
```

pluck('count','status') によって状態をキーにした配列が返されます。便利です。

## すべての状態が表示されるようにる（０個の状態も含め）

グループ別クエリでは 0 件の状態は省かれます。
0 件も含めてすべての状態を取得するには、基準となる状態のリストを用意してそのリストに個数を紐づけます。

アイデアの状態に enum を使うのであれば：

```php
// app/Enums/IdeaStatus.php
enum IdeaStatus: string {
    case Pending = 'pending';
    case InProgress = 'in_progress';
    case Completed = 'completed';
}
```

enum に定義した状態の値を取得するヘルパーを用意します：

```php
// app/Models/Idea.php (example helper)
public static function statusValues(): array
{
    return array_map(fn($case) => $case->value, \App\Enums\IdeaStatus::cases());
}
```

これを使って全状態がそろった個数のコレクションを構築します：

```php
public static function statusCounts($user = null)
{
    $raw = self::selectRaw('status, count(*) as count')
        ->when($user, fn($q) => $q->where('user_id', $user->id))
        ->groupBy('status')
        ->pluck('count', 'status');

    return collect(self::statusValues())
        ->mapWithKeys(fn($status) => [$status => (int) $raw->get($status, 0)]);
}
```

これにより `statusCounts()` はどんな場合でも `['pending' => 4, 'in_progress' => 1, 'completed' => 0]` のように返します。

enum を使用していなければ、モデル上で statusValues() を `public const STATUSES = ['pending','in_progress','completed'];` のような単なる定数の配列の置き換えます。

## Blade：個数付きのフィルターを配置する

ビュー上に各状態のピルバッジ兼リンクを配置し、それぞれの隣に個数を表示します。
このような感じです：

```html
<!-- resources/views/ideas/index.blade.php -->
<div class="filters">
    @foreach (App\Models\Idea::statusValues() as $value)
        <a href="{{ route('ideas.index', ['status' => $value]) }}"
           class="btn {{ request('status') === $value ? 'btn-active' : 'btn-outlined' }}">
            {{ ucwords(str_replace('_', ' ', $value)) }}
            <span class="ml-2 text-sm">{{ $statusCounts->get($value) }}</span>
        </a>
    @endforeach

    <!-- 別途 "All" ボタンを用意（状態のパラメータはなし） -->
    <a href="{{ route('ideas.index') }}" class="btn {{ request()->has('status') ? 'btn-outlined' : 'btn-active' }}">
        All <span class="ml-2 text-sm">{{ $statusCounts->get('all') ?? auth()->user()->ideas()->count() }}</span>
    </a>
</div>
```

注意：ここでは個数のコレクションに `auth()->user()->ideas()->count()` で取得した "all" を追加しています。

## バリデーションとフィルタリングの保護

ユーザからクエリパラメータ `status` を受け取るのであればバリデーションを行いましょう（`?status=bogus` などを渡されてもおかしくならないように）。
２つの方法があります：

### 1. シンプルな配列チェック：

```php
$values = Idea::statusValues();
$status = request('status');

if ($status && ! in_array($status, $values)) {
    $status = null; // 不正な値は無視する
}
```

### 2. バリデーションを利用：

```php
$request->validate([
    'status' => ['nullable', Rule::in(Idea::statusValues())],
]);
```

どちらの場合でも適切な状態でだけフィルタリングをおこないます。

## モデル内にロジックを移動

複雑なクエリのロジックはモデル内もしくはクエリスコープ内にまとめたほうがいいでしょう。
これまで既に `Idea::statusCounts($user)` と `Idea::statusValues()` を作成しました。
状態によるフィルタリングには、もう一つ、ローカルスコープという方法があります：

```php
// app/Models/Idea.php
public function scopeOfStatus($query, $status)
{
    return $status ? $query->where('status', $status) : $query;
}
```

コントローラでこのように使用します：

```php
$ideas = auth()->user()->ideas()->ofStatus($status)->get();
```

スコープを使うとクエリが読みやすく再利用しやすくなります。

## ちょっとしたコツと注意点

- 現在のユーザに関する結果だけが必要な場合はクエリを user から始めましょう（例：`auth()->user()->ideas()`）― 意図せず全検索することを防ぎます。
- コントローラを簡潔で分かりやすくするために `when()` を使いましょう。
- `pluck('count','status')` を使うとグループ別クエリからキーが割り当てられたマップをうまく取得できます。
- PHP の enum を使う場合は `statusValues()` ヘルパーを用意して、バリデーションや繰り返しで値の配列を取得できるようにしましょう。
- "All" で総数を表示したければ、明示的に計算（例：`auth()->user()->ideas()->count()`）して個数のマップにマージしましょう。

## まとめ

フィルタリングでは主に２つのことを行います：条件に応じてクエリに制約を加え、メタデータ（個数など）を生成します。
条件付きの where 句には　`when()` を、個数の取得には `groupBy` + `pluch` を使用し、結果が 0 件の場合でもすべての状態に結果が割り当てられるようにしましょう。
ロジックはモデル内（スコープ／静的ヘルパー）に移動し、コントローラ／ビューをきれいに保ちます。
次回は：これらを利用してさらに UI の構築を進め、すべてきちんとテストされた状態にしましょう。
