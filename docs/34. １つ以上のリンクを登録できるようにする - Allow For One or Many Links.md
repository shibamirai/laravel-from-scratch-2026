# １つ以上のリンクを登録できるようにする

アイデアにリンクを付与する方法について話しましょう ― シンプルな UX、クライアント側の Alpine.js、そして通常のフォームを使ってどうやってリンクのリストを送り、Laravel でバリデーションしてから保存するのかについてです。

## アプローチ方法

入力欄と "+" ボタンを使って行います。
プラスを押すたびに、現在入力されている URL がリンクのリストに追加され、別の入力欄が現れます。
こうすることで好きなだけリンクを追加することができます。
これとは別に、テキストエリアを使用しリンクを一行ずつ貼り付け、サーバ側でそれを分解するという方法があります ― これでも可能ですが、クライアント側でのバリデーションと見た目が犠牲になります。
前者のアプローチ方法の方が使いやすく UX も良いのでそちらをご紹介しましょう。

## マークアップ方法

基本的な構造はこのようになります：レジェンドのついたフィールドセット、flex row で並べた入力欄と追加ボタン、現在入力されているリンクを表す入力欄の繰り返しです。
送信時の重要なポイントに注目：リンクの各入力欄の名前は links[] という形にしてください。
これでサーバが配列を受け取れるようになります。

Blade と Alpine.js はこのようになります：

```html
<fieldset class="space-y-3" x-data="linkForm()">
  <legend>Links</legend>

  <!-- 入力済みのリンクを表示する（JavaScript で制御） -->
  <template x-for="(link, index) in links" :key="link">
    <div class="flex items-center space-x-2">
      <input type="text" :name="'links[]'" x-model="links[index]" class="input flex-1" />
      <button type="button" @click="remove(index)" aria-label="Remove link" class="btn">
        <!-- アイコン -->
        ✕
      </button>
    </div>
  </template>

  <!-- 新しいリンクの入力欄と追加ボタン -->
  <div class="flex items-center space-x-2">
    <input type="url" id="new-link" placeholder="https://example.com" x-model="newLink"
           class="input flex-1" autocomplete="off" spellcheck="false" />
    <button type="button"
            @click="add()"
            :disabled="!newLink.trim()"
            aria-label="Add new link"
            class="btn">
      +
    </button>
  </div>
</fieldset>
```

補足：type="url" を使って基本的なクライアント側でのバリデーションを使えるようにし、必要なら autocomplete をオフにし、アクセシビリティを考えてアイコンだけのボタンに aria-label を付与しています。

## Alpine.js：データと振る舞い

Alpine のコードはできるだけコンパクトに分かりやすくしようと思います。
newLink と links というプロパティを使い、両端の空白をトリムしてから値をプッシュして入力をクリアし、削除（splice）もできるようにします。

Alpine コンポーネントはこのようになります：

```javascript
function linkForm() {
  return {
    newLink: '',
    links: [],

    add() {
      const url = this.newLink.trim();
      if (!url) return;
      this.links.push(url);
      this.newLink = '';
    },

    remove(index) {
      this.links.splice(index, 1);
    }
  };
}
```

Alpine の属性は直に書く方がよければ、上記の構造を x-data で直接定義してください。

> x-model はユーザの入力に連動させるデータバインディングです。

新規の入力欄と各リンクの入力欄の両方で x-model を使用して、ユーザ入力と同期させるようにしましょう。


## なぜ hidden を使うのか（そしてなぜ入力欄が見えている方がいいとされるのか）

リストを JavaScript で扱う場合でも、それをブラウザからリクエストで送信するには name 属性をもった input 要素が必要です。
links[] と名付けられた input を使うと、送信時にそれらが配列にまとめられ Laravel で扱えるようになります。

それらの input は hidden にする（ユーザが編集できないようにする）ことも通常の目に見える要素にする（ユーザが修正・削除できるようにする）こともできます。
送信前に入力値をいじることができるので、目に見える要素にしたほうがユーザにやさしいです。

hidden にした場合の例：

```html
<input type="hidden" name="links[]" x-model="links[index]" />
```

目に見える要素（編集可能）の場合は上記マークアップのセクションで示したとおりです。

## サーバサイドでのバリデーション

フォームリクエストでは、リンクが配列であること（入力されていれば）と、それぞれ URL 形式であることのバリデーションを行います。

```php
public function rules()
{
    return [
        'links' => ['nullable', 'array'],
        'links.*' => ['nullable', 'url', 'max:255'],
    ];
}
```

これによってデータベースに不正な URL が登録されることを防ぎます。
リンクは必ず入力させるようにしたい場合は、そのようにルールを変更してください。

## モデルに保存する

アイデアモデルで links を配列にキャストするようにすれば、送信されたされた値をそのままアサインして保存できます：

```php
$idea->links = $request->input('links', []);
$idea->save();
```

モデル内でのキャスト：

```php
protected $casts = [
    'links' => 'array',
];
```

## UX の詳細と特殊ケース

リンクを追加するときは、前後の空白を削除して入力欄をクリアします。
newLink.trim() の結果が空であれば追加ボタンを使用不可にします。
空の値をプッシュしないようにするクライアント側でのチェックです。

リンクの削除には、index を引数にして splice を使用します：

```javascript
remove(index) {
  this.links.splice(index, 1);
}
```

Alpine の x-for にキーを与えるときは、インデックスをキーにしないでください（インデックスはアイテムがレンダーされると変わります）。
リンクの文字列自体か決まったIDを使いましょう：:key="link"。

## アクセシビリティとテスト

小さくても重要なアクセシビリティへの対応：アイコンだけのボタンには aria-label 属性をつけ、ラベルを見せたくない入力欄にはスクリーンリーダ専用のラベルを用意し、キーボードユーザが困らないようフォーカスの遷移を整えましょう。

テストへの対応として、自動化テストでクラスや変わりやすい構造に頼らずに UI を操作できるように、要素に分かりやすいセレクタ（私は data-test 属性を使います）をつけましょう：

```html
<input data-test="new-link" x-model="newLink" />
<button data-test="submit-new-link" @click="add()">+</button>
```

ブラウザテストでは data-test="new-link" の要素に入力し追加ボタンをクリック、最後にフォームを送信して、リクエストの中に links => ['https://...'] が含まれることを確認しましょう。

## 簡単なまとめ

以下は重要なのでもう一度言います：

- 入力欄と追加ボタンの小さな UI を使い、Alpine.js でその状態を管理しましょう。
- リンクの入力欄を name="links[]" として、各入力値をまとめて配列として送信できるようにしましょう。
- 追加時にトリミングと入力欄のクリアを行い、フィールドが空なら追加ボタンを押せないようにしましょう。
- サーバ側でのバリデーション：'links' => 'nullable|array' と 'links.*' => 'url|max:255'
- ユーザが編集できるように入力欄を表示し、削除ボタンを追加してインデックスで splice しましょう。
- アクセシビリティへの対応として aria-label と場合によってはスクリーンリーダ専用のラベルを用意し、自動化テストのために data-test 属性を追加しましょう。

これで追加、編集、削除、バリデーション、テストのすべてをカバーしました。
