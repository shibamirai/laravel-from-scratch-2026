# アイデアの個別表示

個別のアイデアを表示するページの構築について話しましょう ― 詳細ページです。
コントローラに紐づけ、Blade ビューを作成し、タイトルと詳細を表示し、アイコンによるナビゲーションを追加し、削除を実装し、リンクをつけます。
話をシンプルにするために、画像や手順や編集モーダルは次回に回します。

## show アクションとルート

コントローラにはすでに show アクションを用意して、アイデアインスタンスとともにビューを返すようにしています。
このような感じです：

```php
public function show(Idea $idea)
{
    return view('ideas.show', compact('idea'));
}
```

resources/views/ideas/show.blade.php に Blade テンプレートを作成し、このアクションにリンクするルートで、ルートバインディングによってアイデアインスタンスが自動的にセットされるようにしましょう。

## 基本的な Blade レイアウトとタイトル

はじめはシンプルにしましょう：最大幅とパディングをセットしたラッパー div、タイトルの h1、そしてカードコンポーネント内に詳細を用意します。
このような感じです：

```html
@extends('layouts.app')

@section('content')
<div class="max-w-3xl mx-auto px-6 py-8">
  <h1 class="text-2xl font-bold">{{ $idea->title }}</h1>

  <div class="mt-6 card">
    <p class="text-gray-700">{{ $idea->description }}</p>
  </div>
</div>
@endsection
```

これが基本的なビューです ― タイトルと、詳細を表示するカードです。
シンプルで読みやすいですね。

## フィールドセットとレジェンド

> フィールドセットのレジェンドは入力欄のグループにラベルをつける良い方法です。

後でまとまった入力欄を追加するのであれば（リンク、手順など）、それらをフィールドセットで囲みレジェンドをつけて、スクリーンリーダにも意味的にも明確になるようにしましょう。

## ページ上部のナビゲーション：戻る、編集、削除

コンテンツの上部にナビゲーションを用意して左側にリンク、右側に編集／削除ボタンを置くのが私の好みです。
SVG アイコンを Blade コンポーネントとして保存 (resources/views/components/icons/*.blade.php) し、簡単にスタイリングできるようにしましょう。

アイコンコンポーネントを使った戻るリンクの例です：

```html
<nav class="flex justify-between items-center mb-4">
  <a href="{{ route('ideas.index') }}" class="flex items-center text-sm font-medium gap-2">
    <x-icons.arrow-back class="w-4 h-4" />
    Back To Ideas
  </a>

  <div class="flex items-center gap-2">
    <a href="#" class="btn btn-outline">Edit Idea <x-icons.external class="w-4 h-4 ml-1" /></a>

    <form action="{{ route('ideas.destroy', $idea) }}" method="POST">
      @csrf
      @method('DELETE')
      <button type="submit" class="btn btn-outline text-red-600">Delete</button>
    </form>
  </div>
</nav>
```

delete アクションを用意しておいてください（下記参照）。
アイコンを Blade コンポーネントにするとスタイリングやホバー時の設定をするのが簡単になります。

## Destroy アクション (削除)

コントローラに destroy メソッドを実装して、リクエストを認証し、アイデアを削除し、インデックスにリダイレクトするようにしましょう。

```php
public function destroy(Idea $idea)
{
    $this->authorize('delete', $idea); // 認証のエピソードで紹介したポリシーと紐づける
    $idea->delete();

    return redirect()->route('ideas.index');
}
```

これだけです ― これで上記の削除フォームを実行するとアイデアを削除してリストに戻るようになります。

## 状態とタイムスタンプ

タイトルの下にアイデアの状態と人に分かりやすいタイムスタンプを表示しましょう。
使いまわせるように状態表示はコンポーネントととして取り出します。

```html
<div class="mt-4 flex items-center gap-4 text-sm text-gray-600">
  <x-status-label :status="$idea->status" />
  <span class="text-muted">{{ $idea->updated_at->diffForHumans() }}</span>
</div>
```

直接コンポーネントに状態の値を渡して、コンポーネント内で適切なラベルと色を設定できるようにします。

## リンクセクション

アイデアにリンクが含まれていれば、Links という見出しの下に配置しましょう。
このセクションはリンクが存在するときだけ表示します。

```html
@if($idea->links && count($idea->links))
  <h3 class="mt-6 text-lg font-semibold">Links</h3>

  <div class="mt-2 space-y-2">
    @foreach($idea->links as $link)
      <a href="{{ $link }}" target="_blank" rel="noopener" class="card flex items-center justify-between gap-3 text-green-600 font-medium">
        <span class="truncate">{{ $link }}</span>
        <x-icons.external class="w-4 h-4 text-gray-500" />
      </a>
    @endforeach
  </div>
@endif
```

外部リンクには target="_blank" と rel="noopener" をセットし、サイトから離れることをしらせるアイコンを用意することを考えましょう。

## スタイリングと余白のコツ

余白にはユーティリティクラス（Tailwind）を使いましょう：mt-6 でセクションを分け、上下の間隔は space-y-2 でセットし、flex + gap でインライングループを作ります。
決まった余白をつけてレイアウトに余裕を持たせるのが好みです。
