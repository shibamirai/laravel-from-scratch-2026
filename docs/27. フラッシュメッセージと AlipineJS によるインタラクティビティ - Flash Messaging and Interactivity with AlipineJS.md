# フラッシュメッセージと AlipineJS によるインタラクティビティ

フラッシュメッセージと通知、それから Alpine.js を使ったインタラクティビティのさわりについて話しましょう。
レイアウトにフラッシュメッセージをレンダリングし、Alpine を使ってそれを段階的に拡張して、いい感じにメッセージが自動で消えるようにする手順をお見せしたいと思います。
大変なことはありません ― JS だけで UX を洗練させます。

> フラッシュメッセージとは次のリクエストの間でだけ表示されるメッセージのことです。

重要なポイント：セッションにセットしたフラッシュメッセージ（例えばログイン後など）は、1度表示した後に続くリクエストではセッションから削除されます。

## レイアウトにフラッシュメッセージを配置

セッションコントローラで `session()->flash('success', 'You are now logged in')` のように成功のフラッシュメッセージをセットすれば、レイアウトにそれを表示できます。
Blade にはこのように記述してみましょう。：

```html
<!-- resources/views/layouts/app.blade.php (excerpt) -->
@if (session('success'))
  <div class="fixed bottom-4 right-4 bg-green-500 text-white px-4 py-3 rounded">
    {{ session('success') }}
  </div>
@endif
```

装飾のない素のフラッシュメッセージを右下に配置します。
リダイレクト後に表示されますが、１度しかセッションに残らないので画面を更新すると消えてしまいます。

## インタラクティブ性のために Alpine.js を導入する

UI のちょっとした改善のためだけにわざわざ Vue や React を導入したくないので、Alpine.js を利用します ― モダンで軽く、Blade に素敵な振る舞いをもたらします。
インストールしてフロントエンドに導入しましょう。

```bash
# インストール
npm install alpinejs
```

JS に組み込みます（resources/js/app.js の例）：

```javascript
import Alpine from 'alpinejs';

window.Alpine = Alpine;
Alpine.start();
```

Alpine では x-data, x-text, x-model, x-show, x-init, and x-transition といった小さな部品を使います。
データバインドを行うフレームワークを使った経験があれば馴染みがあるはずです。

## Alpine.js 入門 (簡単な例)

x-data で定義したプロパティはそのコンポーネントで使用することができます。
こんな感じです：

```html
<div x-data="{ greeting: 'hello' }">
  <p x-text="greeting"></p>
  <input x-model="greeting" class="border rounded px-2" />
</div>
```

双方向バインディングによって、input 欄に文字を入力するとすぐにその文字が p タグに反映されます。

ブール値を使って表示のオン／オフを切り替えるのも簡単です：

```html
<div x-data="{ show: true }">
  <p x-show="show">You can see me</p>
  <button @click="show = !show" class="mt-2 px-3 py-1 bg-gray-200 rounded">Toggle</button>
</div>
```

## Alpine.js を使ってフラッシュメッセージを自動で消す

フラッシュメッセージと Alpine を組み合わせて、数秒後にメッセージが消えるようにしましょう。
Alpine が起動する前にフラッシュメッセージが見えてしまわないように、ルートに `style="display: none;"` をセットしてから Alpine に制御を任せましょう。

このようにします：

```html
@if (session('success'))
  <div
    x-data="{ show: true }"
    x-init="setTimeout(() => show = false, 3000)"
    x-show="show"
    x-transition.opacity.duration.300ms
    class="fixed bottom-4 right-4 bg-green-600 text-white px-4 py-3 rounded shadow-lg"
    style="display: none;"
  >
    {{ session('success') }}
  </div>
@endif
```

注意点をいくつか：

- 要素が初期化されると `x-init` が実行されます；ここで３秒後に `show = false` が実行されるようにします。
- `x-show` で要素の表示を切り替えます。
- `x-transition` でフェードをつけます。（フェードの時間は加速は調整できます）
- `style="display: none;"` によって Alpine の起動前にフラッシュメッセージが表示されるのを防ぎます。

遷移をもっと細かく制御したければ、完全な enter/leave 構文が使えます。

```html
x-transition:enter="transition ease-out duration-300"
x-transition:enter-start="opacity-0 translate-y-2"
x-transition:enter-end="opacity-100 translate-y-0"
x-transition:leave="transition ease-in duration-200"
x-transition:leave-start="opacity-100 translate-y-0"
x-transition:leave-end="opacity-0 translate-y-2"
```

## UX について考慮すべきこと

スクリーンリーダーに対応させるのであれば、フラッシュメッセージに明示的に `role` やアクセシビリティ属性を持たせましょう。
ちょっとしたお知らせとして使用する場合はこれで十分です。
ですが重大な通知であれば `aria-live` リージョンを使いましょう。

メッセージをユーザがすぐ消せるようにしたければ、閉じるボタンを追加して `show = false` と紐づけましょう。
もっとユーザのことを考えるのであれば、メッセージにカーソルが置かれている間はタイムアウトをリセットするようにして、読んでる間にメッセージが消えないようにしましょう。

## まとめ

洗練されたフラッシュメッセージを手に入れる手順をお見せしました：サーバ側でメッセージを用意し、クライアント側で Alpine を使ってグレードアップしました。
Alpine は非常に軽く、重厚なフロントエンドフレームワークを導入しなくても一般的なニーズをカバーします。
実際そのカバー範囲は、典型的な Laravel アプリで必要なインタラクティブ性の **85% に及びます**。

必要であれば、レイアウトで再利用できるようにフラッシュメッセージを含めた `x-component` 形式の Blade や、閉じるボタンとホバー中は消えないようにしたものをお見せすることができます。
