# [開発環境の構築](https://laracasts.com/series/laravel-from-scratch-2026/episodes/2)

開発環境を整えて、実際に Laravel で構築を始められるようにしましょう。
私が使っている最も簡単な手順を紹介します：
ヘルパー（Herd）をインストールし、新しい Laravel アプリをスケルトン化し、デフォルトのルート/ビューを覗き、少し遊んでみて慣れることです。
安心してください — 何も壊すことはありません。

## 開発環境の準備

Laravel アプリケーションを作成する前に、基本的な環境を整える必要があります：
PHP、Composer（PHP 依存関係管理ツール）、フロントエンドアセットをコンパイルする場合は Node と NPM、そして Laravel インストーラー（または Laravel CLI ツール）です。
これらが難しそうに思えても心配はいりません — この作業を簡単にしてくれるツールがあります。

> Herd はローカルスタック（PHP、Node など）をインストール・管理する GUI であり、各コンポーネントを手動でインストールすることなくプロジェクトを立ち上げられるようにします。

Herd は初心者や、高速で一貫性のあるローカル環境を求める方に最適です。
PHP バージョン、Node、その他の依存関係をインストールし、各プロジェクトにシンプルなローカルホスト名を提供します。

Herd をインストールした後（ダウンロードし、MacOS ではアプリケーションフォルダにドラッグ）、起動して指示に従いデフォルトのサービスをインストールしてください。
プロジェクトの保存場所とインストールする PHP バージョンを尋ねられます。
ほとんどの学習者にはデフォルト設定で問題ありません。

## Laravel プロジェクトの作成

Herd の起動後、ターミナルを開き Laravel インストーラーを使用してプロジェクトをスケルトン化します。
Herd をインストール済みの場合、PHP とインストーラーは既に PATH に設定されています。

以下が私が使用した基本的な流れです：

```cmd
# 新しい Laravel プロジェクトを作成する（プロジェクトディレクトリ名が後に続きます）
laravel new project-test

cd project-test
```

作成中にツールが追加機能（スターターキット、データベースの選択）について尋ねる場合があります。
学習目的であれば、データベースには SQLite（ファイルベースで設定が最小限）を選択し、依存関係のインストールには「はい」と答えてください。
プロンプトでスターターキットやその他のオプションツールについて言及されても、現時点ではスキップできます。

インストーラーの実行時には、デフォルトのマイグレーション（ユーザー、ジョブなど）も設定されるため、新規アプリを迅速に実行できます。

## アプリを起動してブラウザで確認する

Herd は超簡単です：Herd フォルダ内のプロジェクトはローカルホスト名経由で公開されます。
フォルダ名が project-test の場合、[http://project-test.test](http://project-test.test) にアクセスするとデフォルトの Laravel ウェルカムページが表示されます。
コマンドラインでの操作を好む場合は、以下を実行できます：

```cmd
php artisan serve
# 実行後に http://127.0.0.1:8000 にアクセスする
```

ですが、Herd のホスト名は便利です。
なぜなら、複数のプロジェクトをそれぞれ独自の URL で運用できるです。

## ルートファイルとウェルカムビューを見る

コードを書く前に、エディターでプロジェクトのファイルを開いてください（お好きなもので結構です— VS Code、PhpStorm、Cursor など）。
新しい学習者にフレームワークに慣れてもらうため、私はいつもあるファイルを勧めています：routes/web.php です。

次のような内容が表示されるはずです：

```php
// routes/web.php

use Illuminate\Support\Facades\Route;

Route::get('/', function () {
    return view('welcome');
});
```
これは次のような簡単な英語に見えます：ユーザーがホームページにアクセスしたら、ウェルカムビューを読み込みなさい。

次に resources/views/welcome.blade.php を開き、ごくわずかな変更（単語一つ）を加えてブラウザを再読み込みしてみましょう。
変更が即座に反映されるのがわかります — この編集 → 再読み込みの繰り返しこそが、確実にアプリを築いていく方法です。

## あとがきと次のステップ

私が使っているツール（Herd、Rector、Pint、コードレビュー補助ツールなど）を全てインストールする必要はありません。
ただし Herd はローカル環境を簡単に整える実用的な方法です。
プロジェクトを作成し、Git にコミットし、ウェルカムページが読み込まれることを確認したら、次のステップに進めます。

準備ができたら、次のレッスンに進みましょう。
モデル、マイグレーション、コントローラーのスケルトン作成から始め、最初の機能を一緒に構築します。
ご希望であれば、簡単なチェックリストや私が使用した正確なコマンドシーケンス（Herd、`laravel new`、`php artisan serve`、および簡単な git 操作の数ステップ）を貼り付けることもできます。
