# [最終プロジェクトのセットアップ](https://laracasts.com/series/laravel-from-scratch-2026/episodes/23)

最終プロジェクトについて話しましょう ― これからこれまで学んだことをすべて含んだ小さいけれど本物のアプリを作成します。
実際のワークフローをお見せします：新しく Laravel プロジェクトを作成し、デプロイして、実際の開発で利用しているツールや自動化機能を設定します。
特に面白いことはありません ― 開発を迅速に保守しやすくするために実際に行われる設定です。

## プロジェクト概要

シンプルなアイデア追跡のアプリを作成します。
すぐに理解できるくらい十分小さいですが、認証、バリデーション、通知、状態監視、画像、TODO、マークダウンの解析、フラッシュメッセージ、それからユーザプロファイルについて実践経験を積める十分な内容があります。
私にとっては、Laracast のコースのポテンシャルを紹介する場所です；皆さんにとっては、実際の機能開発で繰り返し参照できるテンプレートです。

アプリには認証、フィルター（完了／実践中／保留）、タイムスタンプ、マークダウン形式の詳細、オプション画像、手順／タスク、リンク、それからユーザプロファイルの編集機能を含める予定です。
これは遊び場です ― さあ始めましょう。

## アプリのセットアップと最初のコミット

プロジェクトを作成してからソース管理を始めるまでの最低限の手順をお見せします。

ターミナルを開いて以下を実行します：

```bash
laravel new idea
cd idea
```

お好きなエディターでプロジェクトを開きます（動画では Cursor を使っています）。
ローカルホストでデフォルトのホームページを確認してください（フォルダ名が DNS になるような環境 (Herd) であれば [http://idea.test](http://idea.test) になるでしょう）。

Git を初期化して最初のコミットします。

```bash
git init
git add .
git commit -m "Initial commit"
```

GitHub にリポジトリを作成してコードをプッシュします（URL はご自身のリポジトリに置き換えます）：

```bash
git remote add origin git@github.com:your-org/idea.git
git push -u origin main
```

## 本番環境にデプロイ (あっという間のデモ)

このライブを手っ取り早く終わらせるために Laravel Forge を使います。
Forge である必要はなくどんなホストでも構いませんが、Forge を使うと簡単です。

Forge では：サーバを作成し、サイトを追加、自分の GitHub リポジトリと接続し、Forge にデプロイさせます（composer インストール、マイグレーション、Nginx の設定までやってくれます）。
今回の場合、ビルドからデプロイまでにかかった時間は 15 秒でした。

## ツールの利用：フォーマット、アップグレード、自動化

アプリの用意ができたら、今後の作業のためにいくつかツールを用意します。

> Pint はオプションのコード整形ツールです。

これは PHP のコードを標準的なスタイルに整形します。
便利なのでコンポーザースクリプトに追加します。

```json
// composer.json (scripts)
{
  "scripts": {
    "format": [
      "vendor/bin/rector --config rector.php",
      "vendor/bin/pint"
    ]
  }
}
```

そして実行します：

```bash
composer run format
```

これはまず Rector を実行（コードのアップグレード/リファクタリングのため）し、続いて Pint でフォーマットします。
コミット手順に組み込んで実行することもできますし、プッシュ前に手動で実行することもできます。

> Rector は PHP コードをスキャンして自動的にアップグレード（型を付与、構文を更新、パターンを修正）するツールです。

インストールして設定ファイルを作成してから Rector を実行し、プロジェクト内のコード全体にルールを適用します。

```bash
composer require --dev rector/rector
vendor/bin/rector process
```

通常は推奨された rector.php の設定をそのままコピーし、そのから必要な設定やルールだけを有効化します。
Rector は、厳密な型やメソッドの戻り値の型、その他近年 PHP で推奨されていることを追加することができます。

## コードレビューと AI の補助

> Code Rabbit はハイレベルな AI によるコードレビューツールで、潜在的な問題や、ベストプラクティスへの改善点を教えてくれます。

私はこれを、コミットに注釈をつけたり、セキュリティ上の問題や属性の付け忘れ、その他ミスしやすい問題を見つけるのに使っています。
利用は任意ですが、セーフティネットとして役立ちます。

> Laravel Boost は、Laravel 開発における AI アシスタントをサポートします ― あなたのプロジェクトの内容（ルート、テスト、ログ、コマンド）を AI に理解させるのを手助けすることで、AI からの提案をより正確なものにします。

artisan コマンドの実行、テストの実行、ルートの検査など、プロジェクト環境を考慮したAIツール機能が必要な場合は、Boost をインストールしてください：

```bash
composer require --dev laravel/boost
php artisan boost:instal
```

インストール後、エディタ拡張機能 / MCP サーバー（Cursor、VS Code など）を有効にしてください。
これにより、AI エージェントが Boost のコンテキストを利用できるようになります。

## エディタ拡張機能と IDE のコツ

エディタに公式の Laravel 拡張機能（PhpStorm なら Laravel plugin、VS Code なら Laravel Extra Intellisense など）をインストールします。
これにより、定義場所への移動、クラスの推測、ルートナビゲーション、Blade アシスタンス、などが行え、生産性が大きく上がります。

## コンポーザースクリプトとショートカット

私は、チームメートがよく使うコマンドを composer.json にスクリプトとして用意するようにしています。

```json
{
  "scripts": {
    "format": [
      "vendor/bin/rector --config rector.php",
      "vendor/bin/pint"
    ],
    "test": "vendor/bin/pest"
  }
}
```

こうしておくといつでも `composer run format` や `composer run test` としてコマンドを実行できるようになります。

## 今日やったことの簡単なまとめ

- `laravel new idea` で Laravel プロジェクトを作成
- Git を初期化してコミットし、GitHub にプッシュ
- サイトにデプロイ（デモでは Forge に）し、アプリを公開
- コード整形／自動化の設定：Pint + Rector、そして `composer run format` のフロー
- 開発用ツールを追加：AI による開発補助のための Code Rabbit + Laravel Boost
- プロジェクトを開発しやすくするための基本的な エディタ／IDE 拡張機能の導入

## まとめと次回

ここでやったことをすべて導入する必要はありません ― 役立ちそうなものだけピックアップしてください。
開発スピードを速めてコードの質を高く保ちたければ、Rector + Pint + litmus CI スクリプトの組み合わせから始めるといいでしょう。
コードレビューと Boost は、大規模プロジェクトにおける”さらなる保険”として重要です。

準備ができたら、次のエピソードから機能の実装を開始しましょう：モデル、マイグレーション、コントローラ、それからアイデアの最初のユーザーフローです。
コーディングを開始しましょう。
